## 一、项目背景与需求分析

### 1.1 选题背景

随着城市化进程加速，城市供水管网规模日益庞大。一个中等城市的地下供水管网包含数百个监测节点，每个节点每小时产生一条流量和水压监测数据。以 500 个监测点、3 年数据量计算，累计数据量超过 **1300 万条**。如何高效存储、分析、可视化这些海量数据，并基于历史数据进行流量预测，是智慧水务建设的核心课题。

本项目模拟城市水利管网监测场景，通过程序生成千万级模拟数据，利用 SpringBoot + MyBatis 框架实现数据的高效管理与分析，并基于线性回归模型实现 24 小时流量预测。

### 1.2 功能需求

| 功能模块 | 描述 |
|----------|------|
| 数据生成 | 模拟程序生成 500 个管网、3 年、逐小时的监测数据 |
| 数据概览 | 总记录数、管网数量、异常占比等关键指标 |
| 流量趋势分析 | 按日期范围展示日总流量/平均流量的变化曲线 |
| 异常状态分析 | 饼图展示正常/轻微/中度/严重异常的占比分布 |
| 高峰时段分析 | 柱状图展示 24 小时各时段的用水量分布 |
| 多管网对比 | 柱状图对比不同管网的流量总量与均值 |
| 工作日/周末对比 | 对比工作日与周末的用水差异 |
| 实时监测面板 | 最新 20 条监测记录列表 |
| AI 流量预测 | 基于历史数据训练线性回归模型，预测未来 24 小时流量 |

### 1.3 数据规模

- **管网数量**：500 个
- **时间跨度**：3 年（1095 天）
- **采样频率**：每小时 1 次
- **总记录数**：500 × 1095 × 24 = **13,140,000 条**

## 四、海量数据生成策略

### 4.1 生成算法

数据生成模拟真实的用水规律，包含以下数学模型：

**基础流量**：`baseFlow = 50 + (pipeIndex mod 50) × 10 + random(0, 30)`

**时段因子**（模拟早晚用水高峰）：

| 时段 | 因子 | 说明 |
|------|------|------|
| 6:00-9:00 | 1.8~2.2 | 早高峰（洗漱、早餐） |
| 11:00-13:00 | 1.3~1.6 | 午间小高峰 |
| 17:00-21:00 | 1.9~2.4 | 晚高峰（晚餐、洗浴） |
| 22:00-4:00 | 0.3~0.5 | 深夜低谷 |
| 其他 | 0.8~1.2 | 正常时段 |

**季节因子**：

| 季节 | 月份 | 因子 |
|------|------|------|
| 夏季 | 6-8月 | 1.3 |
| 冬季 | 12-2月 | 0.8 |
| 春秋季 | 3-5月, 9-11月 | 1.0 |

**周末因子**：工作日 1.0，周末 0.75

**最终流量**：`flow = baseFlow × 时段因子 × 季节因子 × 周末因子 × 随机噪声(0.85~1.15)`

**水压**：流量越大水压略降，符合物理规律

**异常判定**：流量偏离基础值 > 50% 或水压偏离 > 20% 即为异常

### 4.2 批量插入优化

## 六、AI 流量预测模型

### 6.1 模型设计

采用**多元线性回归**模型进行 24 小时流量预测。

**特征维度**：

| 特征 | 说明 |
|------|------|
| hour/24 | 小时归一化值 |
| isMorningPeak | 是否早高峰(6-9点) |
| isNoonPeak | 是否午间(11-13点) |
| isEveningPeak | 是否晚高峰(17-21点) |
| avgFlow/200 | 近30天该小时历史平均流量归一化 |

### 6.2 训练方法

**主方法：正规方程法（最小二乘法）**

$$ W = (X^T X)^{-1} X^T y $$

当矩阵不可逆时，自动降级为**梯度下降法**（学习率 0.01，迭代 5000 轮）。

### 6.3 模型评估

使用 R²（决定系数）评估模型拟合优度：

$$ R^2 = 1 - \frac{\sum(y_i - \hat{y_i})^2}{\sum(y_i - \bar{y})^2} $$

## 七、性能优化设计

### 7.1 COUNT 缓存优化

**问题**：`SELECT COUNT(*) FROM pipe_flow_record` 在 1300 万数据下需全表扫描，耗时 3-5 秒。

**方案**：创建 `table_stats` 缓存表，数据生成完成后写入计数。后续查询直接读取缓存，响应时间从 3 秒降到 **1 毫秒**。

### 7.2 聚合表优化

**问题**：分析类查询（趋势、饼图、对比）需要对千万级原始表做 GROUP BY 聚合，耗时 10-30 秒。

**方案**：预建 `daily_flow_summary`（按天聚合，54.7 万行）和 `hourly_flow_summary`（按小时聚合，1.2 万行）。数据生成完成后通过 INSERT...SELECT 一次性构建，所有分析查询走聚合表。

| 查询 | 优化前扫描行数 | 优化后扫描行数 |
|------|-------------|-------------|
| 日流量趋势 | 480 万 | 18 万 |
| 异常占比 | 480 万 | 54.7 万 |
| 高峰时段 | 480 万 | 1.2 万 |

### 7.3 索引设计

为高频查询字段建立了 5 个二级索引，覆盖单管网查询、日期范围查询、联合查询和异常筛选场景。

## 九、总结与展望

### 9.1 项目总结

本项目成功构建了一个功能完整的城市水利管网流量监测与预测系统，主要成果包括：

1. **海量数据处理**：通过数学建模生成 1314 万条模拟监测数据，覆盖 500 个管网节点、3 年时间跨度
2. **多维数据可视化**：实现了流量趋势曲线、异常占比饼图、高峰时段柱状图、多管网对比图等多种可视化展示
3. **AI 智能预测**：基于线性回归模型实现了 24 小时流量预测，支持 R² 模型评估
4. **性能优化**：通过 COUNT 缓存表、聚合表、索引优化等手段，将大数据查询从 30 秒优化到秒级响应
5. **完整业务流程**：数据生成 → 存储 → 聚合 → 分析 → 展示 → 预测，形成完整闭环

### 9.2 技术亮点

- 使用 SpringBoot + MyBatis 搭建标准 J2EE 架构
- JDBC 批量插入 + 事务优化实现千万级数据高效写入
- 聚合表设计体现数据仓库分层思想
- 纯 Java 实现机器学习模型，无外部依赖
- ECharts 多图表联动，交互体验良好

### 9.3 改进方向

- 引入真实管网 GIS 地理数据，实现地图可视化
- 升级预测模型为 LSTM 或 Prophet 时序模型
- 接入实时数据流实现流式处理
- 增加管网爆管预警等智能报警功能
